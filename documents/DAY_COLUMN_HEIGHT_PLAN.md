# План: Определение технического способа установки высоты day-column

**Дата:** 02.06.2025

**Общая цель:** Установить высоту каждой карточки `day-column` равной 25% от общей высоты видимой области экрана приложения (`100vh`), обеспечивая адаптивное изменение размера и общую прокрутку приложения при переполнении контента, исключая внутреннюю прокрутку самих `day-column`.

## 1. Определение "общей высоты экрана приложения"
Принято, что "общая высота экрана приложения" — это `100vh` (полная высота видимой области браузера).

## 2. Техническое решение для `day-column`

*   **Файл для изменений:** `front/src/components/DayColumn.css`
*   **Для класса `.day-column`:**
    *   Установить `height: 25vh;`
        *   Это свойство будет задавать базовую (минимальную) высоту.
    *   Установить `overflow: visible;` (или удалить существующие `overflow: hidden;` / `overflow-y: auto;`).
        *   Это позволит контенту, превышающему `25vh`, выходить за пределы блока `day-column`, заставляя его растягиваться по высоте содержимого.
    *   Удалить или закомментировать свойство `flex: 1;` (если оно было). Явная высота `25vh` и поведение `overflow: visible` будут управлять размером.
*   **Для класса `.day-events-list` (внутри `.day-column`):**
    *   Удалить или закомментировать `overflow-y: auto;`.
    *   Удалить или закомментировать свойства, связанные с управлением flex-растяжением и минимальной высотой для скролла (например, `flex-grow: 1;`, `min-height: 50px;`). Его высота теперь будет полностью определяться его содержимым.

## 3. Корректировка родительских контейнеров

*   **Файл для изменений:** `front/src/components/TwoColumnWeekLayout.css`
*   **Для классов `.left-section` и `.week-column`:**
    *   Удалить свойство `height: 100vh;`.
    *   Удалить или закомментировать любые свойства `overflow` (например, `overflow-y: hidden;`), так как прокрутка будет общей для страницы.
    *   Они должны оставаться `display: flex; flex-direction: column;` для вертикального расположения `DayColumn`. Их высота будет определяться суммой высот их дочерних элементов.

## 4. Влияние на общую структуру и прокрутку

*   Фиксированные `TopNavigator` и `WeekNavigator` будут занимать свое место.
*   Контейнер `.week-view` (или `#root`/`body`) будет содержать `TwoColumnWeekLayout`.
*   Поскольку `DayColumn` теперь могут растягиваться по высоте своего контента (имея минимальную высоту `25vh`), общая высота содержимого `TwoColumnWeekLayout` может значительно превысить `100vh`.
*   Это приведет к появлению общей вертикальной прокрутки для всей страницы.

## 5. Обработка изменения размеров окна
*   Использование `vh` единиц обеспечит автоматическую адаптацию базовой высоты `DayColumn` при изменении размеров окна браузера.

## 6. Схема взаимодействия компонентов (упрощенно)

```mermaid
graph TD
    Viewport[Viewport (100vh)]
    Viewport --> Body["Body (min-height: 100vh, overflow: auto по умолчанию)"]
    Body --> Root["#root"]
    Root --> WeekView[".week-view (отступы для навигаторов)"]

    WeekView --> TopNav["TopNavigator (fixed)"]
    WeekView --> WeekNav["WeekNavigator (fixed)"]
    WeekView --> TwoColLayout[".two-column-week-layout (высота от контента)"]

    TwoColLayout --> LeftSection[".left-section (высота от контента)"]
    TwoColLayout --> RightColumn[".right-column (высота от контента)"]

    LeftSection --> DayColumn1["DayColumn (height: 25vh, overflow: visible)"]
    DayColumn1 --> DayEventsList1[".day-events-list (высота от контента, overflow: visible)"]
    %% ... остальные DayColumn аналогично ...
```

## Ожидаемый результат от реализации:
*   Каждый `DayColumn` будет иметь минимальную высоту `25vh`.
*   Если контента в `DayColumn` больше, чем на `25vh`, колонка растянется, чтобы показать весь контент.
*   Если контента меньше, `DayColumn` останется высотой `25vh`.
*   Внутренней прокрутки в `DayColumn` не будет.
*   Появится общая прокрутка для всей страницы, если суммарная высота контента превысит высоту видимой области окна.
