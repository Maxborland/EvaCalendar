[x] Задача 1: Разработка мобильной версии (двухколоночный вид)


Описание: Реализовать адаптивный двухколоночный макет для мобильной версии интерфейса. Первая половина недели должна отображаться в одной колонке, вторая — во второй. Обе колонки должны быть доступны для горизонтальной прокрутки/свайпа.

Требования:
Адаптивный дизайн для мобильных устройств (смартфоны, планшеты).
Представление недели в двух основных горизонтально прокручиваемых секциях: "Первая половина недели" и "Вторая половина недели".
Плавный горизонтальный свайп/прокрутка между этими секциями.
Сохранение текущего функционала отображения задач и их взаимодействия.

Критерии приемки:
Приложение корректно отображается на мобильных устройствах.
Неделя разбита на две горизонтально переключаемые секции.
Переключение между секциями происходит плавно.
Вся функциональность, доступная на десктопе (отображение задач, взаимодействие), сохранена на мобильной версии.

[x] Задача 3: Исправление ошибки 404 при перемещении задач


Описание: Исправить ошибку Failed to load resource: the server responded with a status of 404 :3001/tasks/move:1, которая возникает при попытке перемещения задач. Это указывает на некорректный API-эндпоинт или его отсутствие.

Требования:
Проверить URL-адрес эндпоинта /tasks/move на стороне клиента и сервера.
Убедиться, что на сервере существует и корректно обрабатывается эндпоинт для перемещения задач, принимающий необходимые параметры (ID задачи, новые дата/время/позиция).
Проверить логи сервера на предмет ошибок, связанных с этим эндпоинтом.
Убедиться, что клиент отправляет данные в правильном формате.

Критерии приемки:
Ошибка 404 больше не возникает при перемещении задач.
Перемещение задач приводит к успешному обновлению данных на сервере.

Описание: Исправить ошибку NoteField.tsx:27 Error fetching note: SyntaxError: Unexpected token '<', "<!doctype "... is not valid JSON, которая свидетельствует о том, что сервер возвращает HTML вместо ожидаемого JSON для заметок.

Требования:
Идентифицировать причину, по которой сервер возвращает HTML (скорее всего, страница ошибки или редирект), а не JSON.
Проверить эндпоинт, который отвечает за загрузку заметок.
Убедиться, что на сервере корректно настроен Content-Type: application/json для ответов, содержащих заметки.
Проверить логи сервера на предмет ошибок, связанных с загрузкой заметок.

Критерии приемки:
Ошибки SyntaxError: Unexpected token '<' при загрузке заметок больше нет.
Заметки успешно загружаются и отображаются в приложении.

Удалить элемент UI, который представляет собой расчетное поле из модального окна.
Убедиться, что логика расчета заработка (на основе ставки в час и длительности задачи) полностью сохранена и продолжает работать на бэкэнде или в фоновом режиме на фронтэнде (если это не нарушает архитектуру).
Конечный заработок должен по-прежнему сохраняться и использоваться в других частях приложения (например, в отчетах).

Критерии приемки:
В модальном окне больше нет видимого расчетного поля.
Расчет заработка за задачу по-прежнему происходит корректно и сохраняется.
